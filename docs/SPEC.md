# Functional Specification

## Purpose

Static marketing site for **Festibask'Impro**, the improvisation theatre festival in Anglet. Informs visitors about the event (dates, venue, programme, stages, pricing), lineup (Équipe de France, matchs, long-form shows), newsletter signup, and practical/legal info. Primary audience: French-speaking visitors.

## Scope

- **In scope:** Single-page layout with hero (logo, dates, countdown, teaser video, bandes doodles décoratives, liens hero-tags); section “À l’affiche” (spectacles by day: format long + match); témoignages carousel (from JSON, between spectacles and stages); programme (three days, fullscreen mode); stages/ateliers (flip cards, registration/waitlist); tarifs (flip cards); FAQ; à propos; sponsors; waitlist modal (Brevo/Sibforms). Modal « Tenez-moi au courant » (newsletter) retiré ; voir docs/ISSUES.md pour remise éventuelle. legal/contact/footer. Page dédiée **/video/** (watch page) pour la vidéo teaser, afin que Google puisse l'indexer pour les résultats vidéo ; lien « Voir la vidéo teaser » dans le footer.
- **Out of scope:** Ticketing, user accounts, backend CMS, server-side rendering. Pas de routing applicatif ; document principal index.html + page vidéo video/index.html. Google Analytics is commented out in code; Brevo, Sibforms and PostHog (analytics: visites et événements custom) are the active third-party integrations. See [docs/analytics-posthog.md](analytics-posthog.md) for the event catalogue.
- **Mini-jeu Malix :** Jeu enfant autonome (collection de Malix), disponible **uniquement** sous **/malix**. Comportement et périmètre détaillés dans [docs/SPEC-Malix.md](SPEC-Malix.md). Le site principal (index.html, assets/) ne charge rien depuis /malix ; aucun code du jeu dans le bundle principal.

## Main Capabilities

1. **Hero** (et page vidéo) : Logo, dates (15–17 mai), location (Patinoire de la Barre, Anglet), countdown to 2026-05-15 10:30; teaser video (poster by default, play on click). La vidéo teaser est affichée à la place du logo hero, avec le logo du festival en image poster et un ratio 1000/607 (celui du logo). Deux bandes décoratives doodles (images `assets/img/long/doodles-top.avif` au-dessus du bloc vidéo/logo, `assets/img/long/doodles-bottom.avif` sous les liens hero-tags), intégrées dans la même cellule que le bloc vidéo et que les liens. Liens hero-tags : SPECTACLES, MATCHS, STAGES (ancres #valeur, #programme, #stages). Source vidéo : **YouTube** (video ID via `data-hero-video="youtube"` et `data-hero-youtube-id` sur `#heroVideoContainer`) ou **self-hosted** (MP4 depuis `data-src`, poster, sous-titres `assets/video/teaser-festibask-captions.vtt`). En mode YouTube, la vidéo affichée peut être **pilotée par un calendrier** : le fichier `assets/data/hero-video-schedule.json` définit des créneaux (ex. teasers J-3, J-2, J-1, vidéo officielle) avec date/heure de publication ; le script choisit automatiquement la vidéo dont le créneau est passé (dernier `publishAt` ≤ heure du client). En l’absence de calendrier ou avant le premier créneau, le fallback est `data-hero-youtube-id` ou `fallbackYoutubeId` du JSON.
2. **Spectacles (“À l’affiche”):** Day slider (Vendredi / Samedi / Dimanche); for each day: one “format long” block and one “match” block. On mobile and desktop, tap/click on a spectacle banner opens a slider in the block (no fullscreen modal). Match France (Samedi) has multiple slides (EDF intro + players); other spectacles (format long and other matchs) have a single slide (image + details). Slider is closed via the close button or by clicking the block again.
3. **Programme:** Tabs by day; grid of day cards; fullscreen mode; sync with “À l’affiche” day selection. Spectacle slots (format long, match) use solid tinted background (no texture), aligned with tariff cards. Stage slots show instructor image on the right and room label in meta position (like LONG FORM / MATCH); avatar width reduced in normal mode to leave room for the stage title. In fullscreen, clicking the Braquage slot exits fullscreen and opens https://plamarque.github.io/braquage/ in a new tab.
4. **Stages:** Atelier cards (flip for details, instructor bio where applicable); “En savoir plus” / “COMPLET” trigger flip; registration and waitlist buttons open Sibforms modals.
5. **Tarifs:** Pass 1 jour, Pass 3 jours, Stage; flip cards for details. Pour les **pass spectacles** (Vendredi, Samedi, Dimanche, Les 3 jours), le site peut afficher une **alerte de places restantes** ou **« Complet »** lorsque `remaining` ≤ 0 (zéro ou valeur négative, p. ex. dépassement côté tableur), à partir de `assets/data/places-spectacles.json` (généré au build depuis un **CSV publié** Google Sheets ; voir ARCH). Ordre recto (pass spectacles) : titre, prix, lien programme, bouton **« J'achète »** ; la ligne **« ⚠️ plus que XX places! »** (singulier : « ⚠️ plus que 1 place! » ; centrée, sous le bouton) n’est affichée **que si** `remaining` est **strictement inférieur à 100** (et &gt; 0). Si `remaining` ≥ 100, pas de ligne places ; le bouton d’achat reste affiché. Si complet (`remaining` ≤ 0) : le tarif est **barré**, le bouton est remplacé par un bloc **« Complet »** visible (même emplacement, mis en avant), sans achat pour ce pass. **Pass « Les 3 jours » :** affiché **Complet** (même règle visuelle) dès que **l’une au moins** des journées **Vendredi, Samedi ou Dimanche** est complète (`remaining` ≤ 0 pour ce jour dans le JSON), même si le fichier indique encore des places pour le pass 3 jours — on ne vend pas un pass 3 jours si une journée à la carte est épuisée. Les **stages** n’utilisent pas encore ce fichier (slice reportée, [docs/slices/places-stages-ateliers.md](slices/places-stages-ateliers.md)).
6. **Témoignages:** Carousel built from `assets/data/temoignages.json`; placed on the home page **after** “À l’affiche” and **before** “Stages d’impro”; section hidden if JSON empty or invalid. Fields: name, quote, image; role unless `omitHeader` is true; optional signature, imageAvif128/256. With **one** valid entry, prev/next controls and dot indicators are hidden; with **two or more**, standard carousel navigation (arrows, dots, swipe).
7. **Navigation:** Fixed header on scroll; burger + drawer on mobile; desktop nav links.
8. **Modals:** Waitlist (stages « Ça m'intéresse! ») ; Sibforms CSS/JS loaded on first open. Newsletter modal (notify) retiré (voir ISSUES.md). Spectacle details are shown in a slider within the block (not a modal).
9. **Tooltips:** “Format long” and “Match” explained (desktop: hover; mobile: click popup).
10. **SEO / meta:** Canonical, OpenGraph, Twitter Card, Schema.org Event (2026-05-15 to 2026-05-17, location, offers, organizer La Malice). La page **/video/** a son propre canonical, OpenGraph (dont og:video), Twitter Card et Schema.org VideoObject ; elle est listée dans le sitemap et optionnellement dans sitemap-video.xml. La page /malix a son propre jeu de meta (description, canonical, OpenGraph, Twitter Card, entrée sitemap) pour le référencement et le partage, en cohérence avec l’accueil (voir docs/SPEC-Malix.md § SEO / découvrabilité).
11. **Doodles flottants au scroll:** Pendant le scroll vers le bas, un petit doodle peut apparaître de façon aléatoire, dériver lentement à l’écran (effet « cosmonaute », faible gravité) devant le contenu, et disparaître au clic avec un effet « smash » (écrasement). Un seul doodle flottant à la fois ; désactivé si l’utilisateur a `prefers-reduced-motion: reduce`.

## Behavior

- **Entry:** `index.html` (accueil) et `video/index.html` (page de lecture dédiée pour la vidéo teaser) ; assets depuis `assets/`. Build: `npm run build` → copy (index.html, video/, sitemap, etc.) + PurgeCSS + PostCSS (cssnano) + Terser ; sortie dans `dist/`. Deploy: GitHub Actions → GitHub Pages (`dist/` artifact).
- **Video:** Teaser poster shown by default; play on overlay click. Mode chosen in HTML: **YouTube** (`data-hero-video="youtube"`, `data-hero-youtube-id`) — iframe embed loaded on click; or **self-hosted** (default) — MP4 loaded and played on click, captions from `assets/video/teaser-festibask-captions.vtt` (referenced in HTML). In YouTube mode, which video is shown can be **scheduled** via `assets/data/hero-video-schedule.json` (slots with `publishAt` and `youtubeId`); the script picks the last slot whose time has passed; fallback to `data-hero-youtube-id` or JSON `fallbackYoutubeId` when no schedule or before first slot. To revert to self-hosted, remove or set `data-hero-video` to `"self"` and remove `data-hero-youtube-id`.
- **Data:** Testimonials from `assets/data/temoignages.json` (array); hero video schedule from `assets/data/hero-video-schedule.json` (optional); **places pass spectacles** from `assets/data/places-spectacles.json` (objet `passes` avec `remaining` entier par clé `pass-vendredi`, etc. ; fichier par défaut sans chiffres si pas de CSV au build) ; EDF players and spectacle pitches hardcoded in `script.js`. No backend API at runtime.
- **Doodles flottants:** Apparition aléatoire au scroll (scroll vers le bas), dérive lente (physique type cosmonaute), clic sur le doodle = animation smash puis suppression ; désactivés si `prefers-reduced-motion: reduce`.

## Boundaries

- **Inputs:** User visits URL; testimonials from local JSON; form submissions to Brevo/Sibforms (external).
- **Outputs:** HTML/CSS/JS and static assets served to the browser; form data sent to Brevo.
- **External dependencies:** Brevo (tracker + forms), Sibforms (form UI), PostHog (analytics, snippet in index.html; ingestion via reverse proxy e.festibask-impro.fr, ui_host EU; see [docs/analytics-posthog.md](analytics-posthog.md) for event catalogue), Google Fonts (Hubot Sans). [UNCERTAIN] Apple touch icon path (root vs assets/favicon) may vary by deploy.
- **Charte graphique:** Typography and colors are aligned with the Festi'Bask Impro graphic charter: Hubot Sans (Titre 1/2/3, Text, Description) and a 6-color palette (navy, blue-charte, cyan; coral, orange, pink). Variables and usage are documented in docs/slices/charte-graphique.md.
